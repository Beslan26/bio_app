### Medical Assistant Ecosystem (В разработке)

Интеллектуальная медицинская экосистема с открытой модульной архитектурой. Проект разрабатывается как ИТ-решение для сферы здравоохранения (Digital Health) и призван стать цифровым ассистентом, который автоматизирует рутину, помогает пациентам придерживаться плана лечения, а врачам — точнее мониторить состояние здоровья людей.

> **Статус проекта:** Активная стадия разработки. Архитектура строится на базе изолированных Docker-контейнеров. Бэкенд-ядро (FastAPI) находится на этапе реализации эндпоинтов, параллельно разворачивается контейнер Frontend-приложения для интерфейса пользователей.

---
### Смысл и ключевые возможности

Главная цель экосистемы — повысить качество взаимодействия между врачом и пациентом с помощью технологий искусственного интеллекта и автоматизации.

Что реализуется в рамках платформы:

- **Цифровая разгрузка врачей (AI-анамнез):** Пациент может надиктовать свои жалобы голосом. Система преобразует аудио в текст (Speech-to-Text), выделит ключевые медицинские факты, сформирует для врача лаконичную выжимку (саммари) и предварительно подскажет, к какому специалисту стоит обратиться.
- **Интерактивный дневник здоровья и комплаентность:** Врач может назначать пациенту регулярные задачи (прием медикаментов, замеры давления или пульса). Пациент отмечает выполнение в своем интерфейсе, а система строит наглядные графики для отслеживания динамики.
- **Электронная медицинская карта (ЭМК):** Централизованное и структурированное хранение истории болезней, результатов лабораторных анализов и медицинских документов.
- **Инструменты координации:** Гибкое управление графиком приема врачей и возможность оперативной онлайн-записи.

---

### Архитектура и масштабируемость

Проект разрабатывается с прицелом на слабую связанность компонентов (Loose Coupling), что позволяет легко развивать систему. Каждый элемент изолирован в своем окружении:

1. **Frontend App (В процессе разработки):** Пользовательские интерфейсы для двух главных ролей — Личного кабинета Пациента и Рабочего места Врача.
2. **Backend Core API (FastAPI):** Асинхронное ядро системы. Отвечает за обработку медицинской логики, разграничение прав доступа и предоставляет документированный REST API.
3. **AI Service Modul:** Изолированный блок для ресурсоемких задач обработки естественного языка (NLP) и речи — распознавание симптомов, классификация и подготовка медицинских выжатых данных.
4. **Аналитический модуль (Future-proof):** Архитектура закладывается с возможностью подключения продвинутых систем визуализации медицинских метрик (например, open-source BI-инструментов вроде Apache Superset) для детального анализа графиков выздоровления.

### Технологический стек

- **Контейнеризация:** Docker, Docker Compose
- **Frontend:** Vite + TypeScript
- **Backend Framework:** Python 3.11+ / FastAPI
- **ORM & Миграции:** SQLAlchemy 2.0 (Async) / Alembic
- **База данных:** PostgreSQL
- **Безопасность данных:** JWT (Access/Refresh Tokens), Pydantic.
